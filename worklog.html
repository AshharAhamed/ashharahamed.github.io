Learning Project #1

Build a RAG API with FastAPI

My first hands-on AI project: implementing a Retrieval-Augmented Generation (RAG) chat endpoint to understand how modern AI chat applications work end-to-end - from storing documents as embeddings in a vector database, to retrieving relevant context, to generating grounded responses with an LLM.

What I built: Started with a manual RAG walkthrough (retrieval, augmentation, generation) to understand the concept before writing code. Then built a personal knowledge base by chunking profile documents, generating embeddings, and storing them in ChromaDB. Exposed an /ask FastAPI endpoint that converts a question into an embedding, retrieves the most relevant chunks via semantic similarity search, and passes them as context to an LLM (Qwen2.5:0.5B via Ollama) to generate a grounded answer - tested through Swagger UI. Extended the project further with a POST /documents endpoint and metadata filtering to support multiple user profiles (multi-tenancy), so responses stay personalized per user.

Key concepts learned: RAG architecture (retrieval, augmentation, generation), embeddings vs. LLMs, semantic search, vector similarity, metadata filtering, and multi-tenant context isolation.

Challenges & wins: Took about 4 hours end-to-end. The trickiest part was building an intuition for what embeddings actually represent and how similarity search finds relevant chunks - once that clicked, wiring up the FastAPI endpoint and multi-user filtering came together quickly. Next up: going deeper into AI/LLM integration patterns.
